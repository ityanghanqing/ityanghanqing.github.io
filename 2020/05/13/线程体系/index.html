线程体系学习

并行与并发:

并行: 指两个或多个事件在同一个时刻发生
并发: 指两个或多个事件在同一个时间段内发生

进程与线程

 进程: 指在一个内存中运行的程序,每个进程都有一个独立的内存空间 ,一个应用程序可以同时运行多个进程,进程也是程序的一次执行过程,是系统运行程序的基本单位,系统运行一个程序也就是一个进程从创建 ,运行到消亡的过程;
 线程: 进程中的一个执行单元,负责当前进程中程序的执行,一个进程中至少有一个线程,一个进程中可以有多个线程的;

进程与线程的区别:

 进程是有独立的内存空间,进程中的数据存放空间(堆空间和栈空间)是独立的,至少有一个线程;
 线程: 堆空间是共享的,栈空间是独立的,线程消耗的资源比进程小的多

创建线程的常用方式

创建线程的常用方式:
 一 :1.自定义Thread类的子类,也就是继承Thread类,并重写run()方法,该run方法的方法体就代表了线程需要完成的任务,因此把run()方法称为线程执行体
2.创建Thread子类的实例,即创建了线程对象
3.调用线程对象的start()方法来启动该线程
 二:实现Runnable接口,重写里面唯一的run()方法,具体看ImpRunnableTest类详解 

package com.ityang;

/**
 * @Author Qing-俺物语·杨
 * @Date 2020/5/12 22:42
 * @Version 1.0
 */

/**
 * 第二种：
 *   实现Runnable接口方式使用关键字implements实现Runnable接口，重写里面的run方法即可
 *   详细步骤：
 *      1.定义Runnable接口的实现类，并重写该接口的run（）方法，该run（）方法的方法体同样是该线程的线程执行体
 *      2.创建Runnable实现类的实例，并以此实例作为Thread的tareget来创建Thread对象，该Thread对象才是真正的线程对象
 *      3.调用线程对象的start（）方法来启动线程
 */
public class ImpRunnableTest implements Runnable {
    @Override
    public void run() {
        for (int i = 0; i <10 ; i++) {
            System.out.println(Thread.currentThread().getName() + " " + i);

        }
    }
}

继承Thread类与实现Runnable接口区别:

 1.最大的区别就是Runnable接口可以实现线程之间的资源共享,而Thread类不适合资源贡献
 2.避免了java中的单继承所带来的局限性;
 3.线程池中只能放入实现Runnable或Callable类的线程,不能直接方法继承Threa类在启动多线程的时候,需要先通过Thread类的构造方法Thread(Runnable target)构造出对象,然后调用Thread对象的start()方法来运行多线程的代码实际上所有的多线程代码都是通过运行Thread的start()方法来运行的,因此,不管继承Thread类还是实现Runnable接口来实现多线程,最终还是通过Thread的对象的API来控制线程的,熟悉Thread类的API是进行多线程编程的基础;

start方法与run方法的区别:

 1.start()方法:用start()方法来启动线程,是真正实现了多线程,通过调用Thread类的start()方法来启动一个线程,此线程处于就绪(可运行)状态,并没有运行,一旦的到cpu时间片,就开始执行run()方法,但要注意的是,此时无需等待run()方法执行完毕,即可继续执行下面的代码,所以run()方法没有实现多线程
 2.run()方法只是类的一个普通方法,如果直接调用run方法,程序中依然只有主线程这一个线程,其程序执行路径还是只有一条,还是要顺序执行,还是要等待run方法体执行完毕后才可继续执行下面的代码

线程的状态

线程的状态:
 1.创建(初始状态):
实现Runnable接口或者继承Thread类得到一个线程类, new一个实例,线程就进入了初始状态;
 2可运行(就绪):
也就是调用线程对象的start()方法 ,准备就绪,就等cpu给自己资源;
 3.运行:当线程获得CPU时间,线程才从可运行(就绪)进入到运行状态
 4.锁阻塞:当一个线程试图获取一个对象锁,而该对象锁被其他的线程持有,则该线程进入Blocked状态,当该线程持有锁时,该线程将变成运行状态
 5.无限等待:一个线程在等待另一个线程执行一个动作时,该线程进入Waiting状态,进入这个状态是不能自动唤醒的,必须等待另一个线程调用notify或者notifyAll方法才能够唤醒
 6.计时等待:也是进入waiting状态,带有超时参数的常用方法有Thread.sleep,Object.wait
 7.被终止(死亡):因为run方法正常退出而终止,或者因为没有捕获的异常终止了run方法而死亡

sleep与wait区别:

sleep与wait的区别:
 1.来自不同的类 sleep方法是Thread类中的,wait是Object中的
 2.最主要的就是sleep方法没有释放锁,而wait方法释放了锁,使得其他线程可以使用同步控制块或者方法
 3.wait,notify 和notifyAll只能在同步控制方法或者同步控制块了面使用,而sleep可以在任何地方使用
 4.sleep必须捕获异常,而wait notify notifyAll不需要捕获异常

匿名内部类方式实现多线程:

package com.ityang;

/**
 * @Author Qing-俺物语·杨
 * @Date 2020/5/13 8:36
 * @Version 1.0
 */
/**
 * 匿名内部类的方式实现线程
 * 使用匿名内部类的方式实现Runnable接口,重写Runnable接口中的run方法
 */
public class AnonymiytInnerThreadTtest {
    public static void main(String[] args) {
        /**
         *
         *
         *     new Runnable(){
         *         public void run(){
         *             for(int i = 0;i<20;i++){
         *                 System.out.println("张三"+i);
         *             }
         *         }
         *
         *     }
         */

        //上面这个整体相当于  匿名内部类方式
        new Thread(new Runnable() {
            @Override
            public void run() {
                for (int i = 0; i < 20; i++) {

                    System.out.println("张三" + i);
                }
            }
        }).start();


        //使用Lambda表达式:
        new Thread(() -> System.out.println("张三")).start();

    }

}

线程池

引申出问题:我们使用线程的时候就去创建一个线程,这样实现起来非常简单,但就会出现一个问题,如果并发的线程数量很多,并且每个线程都是执行一个时间很短的任务就结束了,这样频繁创建线程就会大大降低系统的效率,因为频繁创建线程和销毁线程需要时间和资源,那么就有一种方法使得线程可以复用,就是执行完一个任务,并不被销毁,而是可以继续执行其他任务

线程池:其实就是一个容纳多个线程的容器,其中的线程可以反复使用,省去了频繁创建线程对象的操作,无需反复创建线程或销毁线程而消耗过多资源其中Executor是跟接口 最主要的接口是ExecutorService接口

创建线程池的四种方式: 是Executors提供的
 一: newCachedThreadPool创建一个可缓存线程池,如果线程池长度超过处理需要,可灵活回收空闲线程,如果无可回收,则新建线程
 二: newFixedThreadPool 创建一个定长线程池,可控制线程最大并发数,超出的线程会在队列中等待
 三: newScheduledThreadPool:创建一个定长线程池 支持定时及周期性任务执行
 四:newSingleThreadExecutor:创建一个单线程化的线程池,它只会用唯一的工作线程来执行任务 保证所有任务按照指定顺序(FIFO,LIFO优先级)执行;
