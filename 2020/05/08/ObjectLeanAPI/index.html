发表于 2020-05-08 | 分类于 Java

Object类学习：

它是类层次结构的根类,也可以叫做所有类的父类或者超类,所有对象(包括数组)都实现了这个类中的方法,我们所写的类也都直接或间接的继承自Object类;

Object类中的常见方法:

equals()方法:

底层源码: public boolean equals(Object obj){
return(this==obj);
}
直译来说就是某个对象是否与此对象相等,默认方法比较的是对象的引用是否相同,由于比较对象的引用对于实际开发中没有意义,所以一般要重写;

引申出一个常见面试题: “==”与”equals()”方法的区别

解答: “==” 是一个比较运算符,既可以比较基本数据类型,也可以比较引用数据类型,比较基本数据类型时比较的是”值”,引用数据类型比较的是地址值
equals()它是一个方法,只能比较引用数据类型,所有的对象都会继承Object类中的方法,若没有重写该类中的equals方法时,equals方法和==号比较引用数据类型无区别,重写后的equals方法比较的是对象中的属性(或者是内容是否一样)

hashCode()方法:

public int hashCode();
返回的是该对象的哈希码值,默认情况下,该方法会根据对象的地址值来计算

引申面试题 为什么重写equals方法还必须重写hashCode方法?

解答: 可以说是一种规定,必须这样做,这样做的目的也就是为了提高效率,采取重写hashCode方法,首先先进行hashCode比较,如果哈希码值不同,那么就没必要进行equals方法的比较了,这样就大大减少了equals方法比较的次数了,这对需要比较数量很大时的效率提高是很明显的,很好的例子就是在集合中的使用,这样又会牵扯到另一个常问面试题,那就是Set是如何去重的?那么肯定就与hashCode方法与equals方法有关系了;

我们知道的是java中List集合是有索引,有序 并且可重复的,而Set是无索引 无序 不可重复的,那么,如何保证不能被放入重复的元素呢,仅靠equals一样方法比较的话,如果数据量很大的话,比如说 一开始就有20000个元素了,那么放入20001个元素时,还需要将其前面的所有元素都进行equals方法再比较一次吗,那么这样的话效率是很低的,但是重写了hashCode方法后.(也是用了算法) 返回一个哈希码值,如果定义进来的数据只要看对应的hashCode值是否有相同的,若有,就使用equals方法再比较,没有就直接插入,大大减少了equals方法的比较,效率指定提高了很多;

简单总结来说就是,hashCode值相同的不一定是同一个对象,要再去比较equals方法,但是如果hashCode值不同,那么肯定不是同一个对象,就不需要使用equals方法比较了;
而equals比较是相同的,那么肯定是同一个对象,但位了效率,也是规范,再重写equals方法时,也必须要重写hashCode方法;

toString()方法:

public String toString(){
return  getClass().getName()+"@"+Integer.toHexString(hashCode));
}//返回该对象的字符串表示,但前提是首先重写Object中的toString方法;

getClass()方法:

public final Class getClass()
//返回Object的运行时类,通常用在反射

综上总结:

当通过一个对象调用Object类中的方法时,会先看自己有没有这个方法,若没有,那么就会向上找父类,一直找到Object类中(因为由上所说就是Object是所有类的父类,所有类都直接或间接的继承自Object,若我们在一个类中没有使用extends继承其它类,那么默认的也是继承自Object的)
