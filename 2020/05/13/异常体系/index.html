异常体系

发表于 2020-05-13 | 分类于 JavaSe

Java异常体系学习总结:

Throwable:所有错误和异常的父类

Error

Throwable下的一个子类,它是严重错误,无法通过处理的错误,只能事先避免

Exception

异常:
 Exception:表示异常,异常产生后,程序员可以通过代码的方式纠正,使程序继续运行,必须要处理的;

异常分类:

 checkd编译时异常 在编译时期,就会检查,如果没有处理异常,则编译失败 例如 io异常 日期格式化异常
 runtime运行时异常,在运行时期,检查异常,例如,数学异常,数组越界异常,空指针异常

异常的处理

关键字: try catch finally throw throws

throw 与throws区别:

 throw出现在函数体中,throws出现在方法函数头
 throw抛出异常,执行throw则一定抛出了某种异常对象,throws表示出现异常的一种可能性,并不一定发生这些异常;
在子类继承父类中,如果父类中没有声明异常,则子类一定不能声明,如果父类中有声明异常,但子类声明的异常一定不能大于父类声明的异常;
